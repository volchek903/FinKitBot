# FinKit Score Bot

Telegram-бот мониторит страницу FinKit `https://finkit.by/app/invest-manually` и отправляет уведомления в личные сообщения только тем пользователям, которые активировали бота через `/start` или `/settings` в ЛС. Каждый пользователь настраивает свои фильтры через кнопки в `/settings`: скор, сумма, срок, ставка, рейтинг, подтверждение дохода, отсутствие исполнительных производств и возрастную группу. После нажатия `Начать работу` пользователю открывается бесплатный период, в течение которого бот проверяет предложения каждую минуту и присылает только подходящие варианты по его настройкам. Длительность периода по умолчанию задается через `TRIAL_DURATION_HOURS`, а администратор может изменить ее в админке в днях. Бот только читает данные и отправляет уведомления: он не инвестирует автоматически, не отправляет заявки, не меняет данные на сайте и не обходит капчу, 2FA или другие защитные механизмы.

## Как создать Telegram-бота

1. Откройте Telegram и найдите `@BotFather`.
2. Отправьте команду `/newbot`.
3. Укажите имя и username бота.
4. Скопируйте токен и вставьте его в `.env` как `TELEGRAM_BOT_TOKEN`.

## Как узнать TELEGRAM_CHAT_ID

`TELEGRAM_CHAT_ID` теперь необязателен для обычной работы бота. Он нужен только если вы хотите:

1. получать служебные сообщения об ошибках в отдельный чат;
2. оставить административный доступ по `chat_id`, если `TELEGRAM_ALLOWED_USER_IDS` пустой.

Чтобы узнать `chat_id`:

1. Напишите любое сообщение своему боту.
2. Откройте в браузере:

```text
https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/getUpdates
```

3. Найдите поле `chat.id` и перенесите значение в `.env` как `TELEGRAM_CHAT_ID`.

Для личного чата обычно используется положительное число. Для группы значение часто отрицательное.

Также можно использовать helper-скрипт:

```bash
python scripts/get_chat_id.py
```

Скрипт читает `TELEGRAM_BOT_TOKEN` из `.env`, ждет новое сообщение боту и печатает строку вида `TELEGRAM_CHAT_ID=123456789`.

## Настройка .env

Создайте файл `.env` из примера:

```bash
cp .env.example .env
```

Заполните обязательные значения:

```text
FINKIT_LOGIN=your-login
FINKIT_PASSWORD=your-password
TELEGRAM_BOT_TOKEN=123456:telegram-token
```

Рекомендуемые настройки для текущего сценария:

```text
DEFAULT_SCORE_THRESHOLD=65
SCORE_COMPARE_MODE=gte
CHECK_INTERVAL_SECONDS=60
TRIAL_DURATION_HOURS=24
TRIAL_MANAGER_CONTACT=@your_manager
```

`SCORE_COMPARE_MODE=gte` означает `score >= threshold`, `SCORE_COMPARE_MODE=gt` означает `score > threshold`. По умолчанию используется `gte`.

Дополнительные параметры:

- `TELEGRAM_ALLOWED_USER_IDS` - список Telegram user ID через запятую для менеджера или администратора. Этим пользователям будут доступны команды `/threshold`, `/check` и `/chat_id`.
- `TELEGRAM_CHAT_ID` - чат для служебных сообщений об ошибках.
- Пользователи из `TELEGRAM_ALLOWED_USER_IDS` могут открыть админку только через `/admin`. Для остальных пользователей бот на `/admin` не отвечает.
- `TRIAL_MANAGER_CONTACT` - текст, который бот покажет после окончания бесплатного периода.

## Команды

`/start` - приветствие и меню с текущим персональным порогом.

`/settings` - изменить персональные фильтры через кнопки и запустить поиск.

`/status` - статус бесплатного периода, основные фильтры и время последней проверки.

`/last` - показать последние 5 найденных предложений.

`/admin` - закрытая админка: статистика пользователей, рассылки, список пользователей с пагинацией.

`/help` - показать список команд.

Административные команды:

- `/admin` - открыть админ-панель. Доступ есть только у Telegram user ID из `TELEGRAM_ALLOWED_USER_IDS`.
- `/threshold 70` - изменить порог скор балла. Значение должно быть числом от 0 до 100.
- `/check` - запустить проверку вручную.
- `/chat_id` - показать ID текущего чата. Команда отвечает только пользователям из `TELEGRAM_ALLOWED_USER_IDS`.

## Админка

Команда `/admin` открывает inline-панель с основными разделами:

1. статистика пользователей: регистрации за 24 часа, 7 дней, 30 дней и за все время, а также текущие статусы доступа;
2. тестовый период: ввод количества дней для бесплатного доступа; значение применяется к новым и уже активированным пользователям от даты их активации;
3. рассылки: одно сообщение всем пользователям из базы или конкретному пользователю по его Telegram ID;
4. список пользователей: пагинация по 10 пользователей на страницу со статусом, датами и текущими фильтрами каждого пользователя.

## Как работает доступ

1. Пользователь пишет `/start` или `/settings` в личные сообщения боту.
2. Бот показывает клавиатуру с отдельной динамической кнопкой под каждый фильтр и кнопкой `Начать работу`.
3. Пользователь нажимает нужную кнопку, вводит значение, и бот сохраняет его персонально для этого пользователя.
4. Доступны фильтры: `amount_min`, `amount_max`, `term_min`, `term_max`, `interest_rate_min`, `interest_rate_max`, `borrower_rating_min`, `borrower_rating_max`, `borrower_score_min`, `borrower_score_max`, `borrower_income_confirmed`, `borrower_enforcement_up_to_1_month_absent`, `borrower_age_group`.
5. После нажатия `Начать работу` бот активирует бесплатный период на срок из текущих настроек и сразу запускает первую проверку.
6. Пока период активен, бот проверяет FinKit каждую минуту и отправляет подходящие предложения только в личные сообщения пользователя с учетом всех его фильтров.
7. После истечения срока бот присылает сообщение о завершении бесплатного периода и просит написать менеджеру.

## Как бот получает предложения

1. Открывает страницу FinKit через Playwright.
2. При необходимости авторизуется с логином и паролем из `.env`.
3. Сохраняет session state в `PLAYWRIGHT_STATE_PATH` и использует его при следующих запусках.
4. Перехватывает JSON-ответы от `https://api-p2p.finkit.by`.
5. Если найден payload с предложениями, использует API-данные и пытается обработать пагинацию по `next`, `page/per_page/total` или `offset/limit/total`.
6. Если API не найден, парсит DOM-таблицу и нажимает доступную кнопку следующей страницы в paginator, пока страницы не закончатся.

В логах видно источник данных: `source=api` или `source=dom`.

## Ограничения

- Бот не инвестирует автоматически и не нажимает кнопки инвестирования.
- Бот не обходит капчу, 2FA, SMS-коды и другие защитные механизмы.
- Частоту проверок нужно держать разумной. Для сценария с пользователями по умолчанию используется `CHECK_INTERVAL_SECONDS=60`.
- Если FinKit меняет структуру страницы или API, может потребоваться обновить парсер.
